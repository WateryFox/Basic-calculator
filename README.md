I developed this project to master the different ways functions can be used to organize and simplify code. The goal was to move beyond basic scripts and explore the four main types of functions: those with or without parameters, and those that either return a value or simply perform an action (void). By rebuilding a calculator using these principles, I practiced how to pass data between different parts of a program, ensuring that the input, the operation, and the final calculation are handled by dedicated, independent blocks of logic.

One of the most important concepts I practiced in this script was the difference between Local and Global variable scope. I experimented with the global keyword to understand how a function can modify a variable that exists outside its own block. This taught me about data persistence and the risks of "side effects" where changing a variable in one function might accidentally break another. Mastering this distinction is vital for writing clean code that doesn't have hidden bugs when the project gets larger.

The technical highlight of the script is the "Advanced Calculator," which uses a modular design to handle user interaction. I broke the process into three distinct phases: a start() function to get the operator, an insert_num() function to collect the data, and a calculate() function to process the math. This design allowed me to practice using Tuples to return multiple values at once from a single function call. By separating the "Input" logic from the "Calculation" logic, I made the code much easier to read and debug.

Through this exercise, I learned how to use return statements to "hand off" results from one function to the next. This is the foundation of Scalability; for example, if I wanted to add more complex math like exponents or roots, I would only need to update the calculate function without changing how the user enters their numbers. It’s a great example of how functional programming makes code more organized and easier to maintain in the long run.
